# Enquadramento do Problema

##  1. Problema: o que dói hoje?

- Pequenas e médias lojas possuem **baixa presença digital**, ficando limitadas ao público físico  
- Consumidores recorrem a marketplaces internacionais, pagando **fretes altos e impostos**, mesmo havendo opções locais  
- Não existe uma plataforma que una lojas regionais com **busca, compra e delivery integrado**

---

## 2. Usuários/personas: quem usa e por quê? 

### Consumidor
- Busca praticidade (comprar sem sair de casa)  
- Quer encontrar opções locais com facilidade  
- Tem receio de comprar roupas online (tamanho, caimento, qualidade)

### Lojista
- Quer aumentar vendas  
- Possui pouca presença digital  
- Precisa de visibilidade sem alto custo de marketing  

### Entregador
- Busca geração de renda  
- Atua com entregas sob demanda  

---

## 3. Objetivos Mensuráveis

- Reduzir o tempo médio de compra para **menos de 5 minutos**  
- Aumentar a visibilidade de lojas locais em **+50% (acessos na plataforma)**  
- Garantir taxa mínima de conversão de **10% nas buscas realizadas**

---

## 4. Escopo do MVP 

Funcionalidades essenciais para validar a proposta:

- Cadastro e autenticação de usuários (consumidor e lojista)  
- Catálogo de produtos (com fotos, descrição, tamanho e preço)  
- Sistema de busca com filtros (categoria, preço, tamanho e proximidade)  
- Carrinho de compras e finalização de pedido  
- Integração com pagamento 
- Notificações de pedidos para lojistas  
- Chat de conversa com lojista 
- Histórico de pedidos  

---

## 5. Fora do Escopo 

Funcionalidades que **não farão parte do MVP**:

- Sistema próprio de delivery completo  
- Modelo automatizado de prova/devolução  
- Chat em tempo real  
- Sistema de avaliações  
- Painel administrativo avançado  
- Inteligência artificial para recomendação  

---

## 6. Restrições

- Plataforma: Mobile + Web 
- Prazo: semestre letivo  
- Stack tecnológica:
  - Mobile: React Native + TypeScript  
  - Web: React + TypeScript  
  - Backend: Java + Spring Boot  
  - Banco de dados: PostgreSQL  
- LGPD:
  - Armazenamento seguro e criptografado de dados sensíveis  
- Integração:
  - API de pagamento (Mercado Pago, Stripe ou similar)  
- Infraestrutura:
  - Docker + deploy em cloud  

---

## 7. Saída da Etapa (Visão do Produto)

Desenvolver uma plataforma que funcione como um **“iFood de roupas”**, conectando consumidores a lojas locais através de:

- Um catálogo centralizado  
- Processo de compra simples e rápido  
- Pagamento integrado  
- Experiência digital acessível
